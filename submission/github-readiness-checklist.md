# GitHub Readiness Checklist

Use this before sending a public repo link.

## Must Check

- `README.md` and `README.en.md` are both readable
- `SKILL.md` reflects the current command set and config model
- no generated outputs are committed
- no private or branded third-party assets are committed unless intended
- no secrets, API keys, `.env`, or personal machine paths are committed
- fixtures are neutral and safe to share

## Repo Story

The repo should be understandable in under 2 minutes:

- what problem it solves
- how PixVerse CLI is used
- what commands exist
- how to try it safely with dry-run or local render

## Best Public Entry Points

Point reviewers to:

- `README.md`
- `README.en.md`
- `SKILL.md`
- `submission/pixverse-cli-skill-submission.md`

## Good Optional Additions

- one screenshot of the terminal workflow
- one sample output MP4 or GIF
- short release note in the repo description

## Suggested Repo Description

```text
Agent-friendly multilingual character video workflow powered by PixVerse CLI and Remotion.
```
